Appliances

  • The commandline client (osc) is a subversion-like tool for the OBS. Use it to edit sources, metadata, trigger builds and gather results.

  • A full fledged web based user interface for the OBS. Your users can access all the OBS functionality right from any browser.

  • An REST based application programming interface (API) for all the OBS server functionality. The clients use this to control the server.

  • The server backend stores all the sources and binaries, schedules builds, controls the workers and generates the repositories.

  • The worker requests all the sources and binaries from the server and then does the actual compiling work.

  • Use these images to deploy a Linux OS (openSUSE) and OBS components to your own physical or virtual server.
